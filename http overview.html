Hypertext Transfer Protocol

What Is HTTP?

Hypertext Transfer Protocol (HTTP) is the underlying technology that allows web servers (like the ones Amazon and Google use to serve their websites) to respond to requests from web browsers (like you, with Chrome or Firefox or whichever browser you use). In other words, HTTP is the foundation for communication on the World Wide Web.

What Do HTTP Conversations Look Like?

First, a client sends a request to a web server. The web server then processes the request and sends a response back to the client, completing the HTTP exchange.

Both the request and the response contain important information in their headers, and often after the headers include a body. The request headers will tell the server if the client has any authentication credentials, what HTTP method (AKA verb) the client is using, metadata about the body (if one is sent), and other relevant data about the client. If a request includes a body, it is typically data that the client wishes the server to store in a database.

The response headers will tell the client if the server was able to successfully process the request (specifically, by returning a 3-digit status code), details about the server and connection, and metadata about the response body (). The body of a response is the information the client requested, often a web page.

HTTP Verbs and Status Codes

In a request, the client uses HTTP verbs to ask a web server to perform specific actions.

The most common verbs are:

  • GET : ask for data from the server.
  • POST : send data to the server for storage.
  • PUT : update data on the server.
  • DELETE : permanently remove data from the server.

An HTTP response will always include a status code, which tells the client how the server processed the request. Status codes are grouped into five categories, with the first digit representing the category:

  • 1xx : informational messages, such as "Continue" or "Switching Protocols".
  • 2xx : successful responses, such as "OK" or "Created".
  • 3xx : redirections, such as "Moved Permanently" or "See Other".
  • 4xx : client errors, such as "Bad Request" or "Unauthorized".
  • 5xx : server errors, such as "Internal Server Error" or "Not Found".
